Mundlak-Zerlegung: Within- und Between-Effekt sauber trennen

13. Juli 2026 · ~7 Min · Nico Schäfer

Wer Paneldaten mit einem simplen Random-Effects-Modell auswertet, mischt zwei Fragen, die nichts miteinander zu tun haben: „Was passiert, wenn eine Einheit sich über die Zeit verändert?" und „Wie unterscheiden sich verschiedene Einheiten?". Der Mundlak-Trick trennt beides — mit einer einzigen zusätzlichen Variable und ohne den Effizienzverlust eines reinen Fixed-Effects-Modells.

Das Problem an einem Beispiel

Angenommen, wir modellieren Streckenstörungen im Schienenverkehr als Funktion der Instandhaltungsintensität. Über mehrere Jahre je Streckenabschnitt (strecke_id × jahr) stellt sich die Frage: Senkt mehr Instandhaltung die Störungen? Ein naives Random-Effects-Modell gibt einen Koeffizienten zurück, der Within- und Between-Variation vermischt — und genau dann verzerrt ist, wenn stärker beanspruchte Strecken sowohl mehr gewartet werden als auch mehr stören. Selektion, als Effekt getarnt.

Die Mundlak-Idee in einem Satz

Nimm zusätzlich den Einheiten-Mittelwert jeder zeitveränderlichen Kovariate ins Modell auf. Der ursprüngliche Koeffizient misst dann den reinen Within-Effekt (Veränderung innerhalb einer Strecke), der Koeffizient des Mittelwerts die Between-Korrektur. Ist letzterer signifikant, war das naive RE-Modell verzerrt — das ist zugleich ein handlicher Hausman-Test.

mundlak.R
# Einheiten-Mittelwert der zeitveränderlichen Kovariate
d <- d |> group_by(strecke_id) |>
       mutate(instand_bar = mean(instandhaltung)) |> ungroup()

# Mundlak-RE-Modell: Within-Effekt + Between-Korrektur
library(plm)
m <- plm(stoerungen ~ instandhaltung + instand_bar,
         data = d, index = c("strecke_id", "jahr"),
         model = "random")

library(lmtest)
coeftest(m)   # instand_bar signifikant  =>  RE war verzerrt

Der Koeffizient auf instandhaltung ist jetzt der Within-Effekt — numerisch identisch zum Fixed-Effects-Schätzer, aber im RE-Rahmen, sodass zeitkonstante Merkmale (Streckentyp, Region) weiter im Modell bleiben dürfen. Der Koeffizient auf instand_bar fängt exakt die Selektion ab, die den naiven Schätzer verzerrt hätte.

Der ehrliche Zusatz: Störungen sind Zähldaten

Das lineare Modell oben erklärt die Mechanik am saubersten. Streckenstörungen sind aber Counts, oft überstreut — methodisch gehört die Mundlak-Korrektur hier in ein (Quasi-)Poisson-Modell mit zufälligem Streckenintercept. Dasselbe Prinzip, andere Familie:

mundlak_count.R
library(glmmTMB)
mc <- glmmTMB(stoerungen ~ instandhaltung + instand_bar + (1 | strecke_id),
             family = nbinom2, data = d)   # Overdispersion abgedeckt

Fazit

Die Mundlak-Zerlegung kostet eine Zeile Code und beantwortet die Frage, an der naive Panelmodelle scheitern: Was ist Wirkung, was ist Selektion? Wer Effekte berichtet, ohne beides getrennt zu haben, berichtet im Zweifel das Falsche.
